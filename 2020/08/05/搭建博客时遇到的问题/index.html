搭建博客时遇到的问题

发表于 2020-08-05 | 分类于 blog | | 阅读次数:
字数统计: 679 | 阅读时长 ≈ 2

1、hexo更换主题后，hexo s在本地查看发现网页乱了，css样式没了（只剩下字，没格式）

解决办法：先清除缓存，再部署网站，建议养成此习惯

- hexo clean

- hexo g

- hexo s

2、搭建完在本地运行没问题，但通过域名访问时却404

解决办法：修改_config.yml文件中的url地址和根目录

- url: 填github pages给我们分配的网址，setting里有个绿勾那里

- root: 填搭建博客的仓库名，setting里的repository name

3、hexo categories和tags页面不显示，网址后出现/20%/

解决办法：修改主题配置文件_config.yml中的menu

- 把 / 后面的空格全部去掉,就是/ ||变成/||

4、github.com 拒绝了我们的连接请求,打不开

解决办法：估计又是墙的问题，修改hosts文件，需要管理员权限

- 1、打开https://link.jianshu.com/?t=http%3A%2F%2Ftool.chinaz.com%2Fdns

- 2、在检测输入栏中输入GitHub官网

- 3、把检测列表的某个ip输入到hosts里，并加上域名

- 修改hosts方法：选中记事本——>右键——>点击“以管理员身份运行”，打开记事本——>在记事本左上角点击文件->打开，在搜索框输C:\Windows\System32\drivers\etc，进入hosts所在目录——>点击右下方所有文件，打开hosts文件，然后进行修改，最后保存并退出即可。

5、git clone 失败 fatal: early EOF fatal: the remote end hung up unexpectedly fatal: index-pack failed

解决办法：最简单的是直接在github上下载你要克隆的仓库的压缩包，然后粘贴到对应的文件夹并解压。

下面说的是网上查的，也可以试一下

- 1、导致这样的原因是SSL错误，克隆远程仓库时，用env命令设置GIT_SSL_NO_VERIFY环境变量为”ture”，并同时调用正常的git clone命令

完整的命令如下：env GIT_SSL_NO_VERIFY=true git clone https://<host_name/git/project.git

- 2、在克隆完毕的仓库中将http.sslVerify设置为”false”

完整的命令如下：git config http.sslVerify “false”

6、博客文章上插入图片

1、把主页配置文件_config.yml 里的post_asset_folder:这个选项设置为true

2、在你的hexo目录下执行这样一句话npm install https://github.com/7ym0n/hexo-asset-image –save，这是下载安装一个可以上传本地图片的插件

3、等待一小段时间后，再运行hexo n "xxxx"来生成md博文时，/source/_posts文件夹内除了xxxx.md文件还有一个同名的文件夹

4、最后在xxxx.md中想引入图片时，先把图片复制到xxxx这个文件夹中，然后插入图片不是用md语法，而是
